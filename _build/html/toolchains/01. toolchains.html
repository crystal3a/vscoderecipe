工具链¶

在软件开发中，工具链（英语：toolchain [1] ）是一系列用于制作软件的工具。这些工具一般一个接一个地运用，一件工具的输出输入至下一件工具，因此得名。但广义上可以单单是指多个相关的工具。

在这份文档中是指广义上的工具链，相近的概念是工具带（英语：tool belt）。如果本文的读者和我一样曾经大量阅读微软出版社的技术书籍的话，那一定会对相关书籍的封面记忆犹新。

我的工具链¶

对一名开发者来说，充分利用各式各样的工具来加快开发进度，是根植在我们心中的坚定信念。作为一名目前使用Python作为主力开发语言的开发者，我现在常用的工具如下，供大家参考。同时我目前基于Windows平台进行开发。

Cmder¶

Cmder 是由 Samuel Vasko 开发的命令行模拟器，用来增强Windows下的命令行功能。在官网 [2] 上可以选择精简版或者完整版进行安装。完整版本中包含了git-for-windows，这意味着你可以在Windows下直接运行Linux命令的移植版本，来实现Linux下相同的操作。当然你也可以安装独立的Git客户端在搭配使用精简版的Cmder。比如下图中显示的就是在Cmder中使用ssh命令链接Azure上运行的Linux虚拟机。同时Cmder也是Django项目开发过程中必不可少的一个工具，在相关章节会涉及到该部分内容。

Git 客户端¶

Git 客户端就不用多介绍，用来进行版本管理的工具，可以从官网 [3] 下载适用于Windows平台的最新版。至于项目托管平台，Github或者国内的coding.net都是不错的选择，如果是公司项目那可以选择自建Gitlab。
